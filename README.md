# 마라톤 일정 트래커

마라톤 대회의 접수 일정과 개최 일정을 한눈에 확인할 수 있는 웹 서비스입니다.  
런린이 남편이 관심 있는 대회의 접수 기간을 놓치지 않도록 만들었습니다.

## 주요 기능

- 마라톤 대회 목록 조회
- 접수 시작일 / 접수 마감일 / 개최일 확인
- 지역, 개최 월, 접수 월, 접수 상태, 거리, 검색어 기반 필터링
- 접수 마감이 임박한 대회 별도 표시
- 리스트 뷰 / 캘린더 뷰 전환
- 관심 대회 즐겨찾기
- 공유 기능
- 공식 사이트 및 접수 페이지 이동

## Backend

백엔드는 FastAPI로 구성되어 있으며, 마라톤 대회 데이터를 크롤링하고 프론트엔드에 API 형태로 제공합니다.

`roadrun.co.kr`의 마라톤 일정 페이지를 크롤링해 대회명, 개최일, 장소, 거리, 공식 URL 등의 기본 정보를 수집합니다. 이후 각 대회의 상세 페이지에 접근해 접수 시작일과 접수 마감일을 추가로 파싱하고, 현재 날짜를 기준으로 접수 예정, 접수 중, 접수 마감, 완료 등의 상태를 계산합니다.

수집한 데이터는 `marathons.json` 파일에 캐싱하며, 서버 시작 시 데이터가 없으면 초기 크롤링을 실행합니다. 또한 APScheduler를 사용해 6시간마다 자동으로 데이터를 갱신합니다.

### API

- `GET /api/marathons`
  - 마라톤 목록 조회
  - 지역, 개최 월, 접수 월, 상태, 거리, 검색어 필터 지원

- `POST /api/crawl`
  - 백그라운드 크롤링 수동 실행

- `GET /api/status`
  - 마지막 업데이트 시간과 전체 데이터 개수 조회

## Frontend

프론트엔드는 React와 Vite로 구성되어 있으며, Tailwind CSS를 사용해 UI를 구현했습니다.

사용자가 페이지에 접속하면 `/api/marathons` API를 호출해 마라톤 데이터를 받아오고, 필터 조건이 변경될 때마다 쿼리 파라미터를 조합해 다시 요청합니다. 받아온 데이터는 카드형 리스트 또는 캘린더 형태로 보여줍니다.

리스트 화면에서는 대회별 개최일, 장소, 거리, 접수 기간, D-day, 접수 상태를 확인할 수 있습니다. 캘린더 화면에서는 접수 시작일과 마감일 기준으로 일정을 배치해 날짜별로 어떤 대회가 열리거나 마감되는지 확인할 수 있습니다.

즐겨찾기 기능은 브라우저 `localStorage`를 사용해 저장합니다.

## Deployment

백엔드는 Railway에 배포했습니다.  
GitHub 저장소와 Railway를 연동해 코드 변경 사항이 반영되면 자동으로 배포되도록 구성했습니다.

Railway에서는 `uvicorn main:app --host 0.0.0.0 --port $PORT` 명령으로 FastAPI 서버를 실행하며, `/api/status`를 헬스체크 경로로 사용합니다.
